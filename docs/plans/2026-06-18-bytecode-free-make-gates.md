# Bytecode-Free Make Gates

## Context

The baseline syntax checker compiles source in memory and no longer writes
bytecode, but the canonical `make check` and `make test` paths invoke unittest
without setting `PYTHONDONTWRITEBYTECODE`. A normal caller with no bytecode
environment variables produces repository-local `__pycache__` directories and
`.pyc` files. Verification should not dirty the worktree by default.

## Prioritized Engineering Tasks

1. **P0: Make canonical gates artifact-free.** Export the no-bytecode runtime
   setting from the Makefile rather than requiring caller configuration.
2. **P1: Prove caller independence.** Run the gate with bytecode-related
   variables explicitly unset and assert no cache artifacts appear.
3. **P1: Preserve every alias.** Keep `check`, `test`, `lint`, and `build`
   behavior aligned without weakening the 31-test suite.
4. **P2: Keep evidence durable.** Add mutation-sensitive static contracts,
   synchronized guidance, and exact hosted verification.

## Requirements

- Set `PYTHONDONTWRITEBYTECODE=1` for every canonical Make target by default.
- Preserve explicit caller overrides only when they remain nonempty and safe;
  the repository default must prevent bytecode creation.
- Add a focused test that unsets `PYTHONDONTWRITEBYTECODE` and
  `PYTHONPYCACHEPREFIX`, invokes the canonical gate in an isolated copy, and
  verifies no `__pycache__`, `.pyc`, or `.pyo` artifact appears.
- Preserve all 31 existing tests and provider response, URL, JSON, template,
  timeout, redirect, and CI boundaries.
- Keep repository-root and external-directory invocation working.
